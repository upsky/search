Шпоргалка по запросам.

GET:
  • ?q=query - послать запроса анализатору
  • ?get=learn_data - получить текущую версию обучающих данных
  • ?get=learn_data&v=2 - получить обучающие данные версии 2
  • ?get=categories - получить текущую структуру категорий
POST:
  • learn_data - обучающие данные в формате json. Формат:
    {
    "version": 0,
    "categories": {
        "Категория 1": {
            "id": 1,
            "learn_data": [
                "Обучающий пример 1.1",
                ...
            ],
            "categories": {
                "Подкатегория 1.1": {
                    "id": 2,
                    "learn_data": [ ... ],
                    "categories": { ... }
                },
                ...
            }
        },
        "Категория 2": {
            ...
        }
    }
}
    В обучающих данных нет обязательных полей. Вот поля, которые есть в принципе:
    • categories - категории, тип dict(мапа), не обязательное поле
    • learn_data - обучающие тексты, тип list(список), не обязательное поле
    • id - id категории, по умолчанию автогенерится, но можно указать, тогда будут юзаться указанные id, лишь бы не было конфликтующих
    • version - это поле автогенерится;
